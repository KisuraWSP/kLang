Quick start

From source file to running program in a few commands.

Build the CLI with Go or download a reviewed native release. Folder projects use first.klang as their entry file, and command-line values arrive through the builtin immutable Args list.

go build -o kLang .
./kLang --help
go run . new examples/myproject
go run . new examples/myproject --entry=["Process","Int"]
go run . check examples/helloworld
go run . run examples/helloworld
go run . run examples/commandlinearena demo 100
go run . check examples/helloworld
go run . fmt examples/helloworld --check
go run . test examples/helloworld
go run . package examples/helloworld --backend=Standalone
go run . package examples/helloworld --backend=JS
go run . serve examples/helloworld --port=8080

Language surface

Expressive types without giving up visible semantics.

The parser, checker, interpreter, formatter, standard library, and code-generation backends share one evolving language model.

Typed bindings

Immutable-by-default variables, explicit mut, typed local/global, inferred let/val/var, strict const, destructuring, discard _, and lazy declarations.

Data and safe fallibility

Use typed collections, deterministic Set, dynamic Table, immutable JSON, and standard Option/Result values with checked access and transforms.

Function-first model

First-class functions, lambdas, callbacks, inline candidates, pass-by-value defaults, explicit ref, named returns, tuple-style multiple returns, defer, and run blocks.

Generics and type construction

Combine restrict[...], named constraints, traits, exact type aliases, and struct-style alias functions with inferred fields, extension methods, and JSON tags.

Metaprogramming surface

Parsable[T], Program, BuildSystem, and WorkSpace expose parsed source, AST data, runtime type information, arguments, and build metadata.

Diagnostics and tooling

Source-aware Context/ErrorContext, canonical formatting, a first-class test runner, runtime reports, stack/state debugging, and source-fingerprint caching support daily work.

Example

A compact kLang program.

This example uses a type alias, an enum, safe option recovery, a typed list, and exhaustive pattern matching.

type user_names = List[String];

enum BuildState {
    Pending;
    Running;
    Complete;
}

function Greeting(name : String) : String {
    return "hello, " + name.uppercase();
}

function Main() {
    local Option[String] requestedName = Some("kLang");
    local String name = option_unwrap_or(requestedName, "friend");
    local user_names contributors = ["Ada", "Rob", name];
    local BuildState state = BuildState.Complete;

    if state == {
        case BuildState.Pending:
            print("waiting");
        case BuildState.Running:
            print("building");
        case BuildState.Complete:
            print(Greeting(name), contributors.count);
    }
}

Build and package

Three backends, each honest about what it does today.

All backends use the shared Check, Emit, and Package contract while targeting different execution environments.

Standalone

Packages checked kLang source and its manifest for the Go interpreter/runtime. Native CLI executables are distributed separately through reviewed GitHub releases.

Native JavaScript

Lowers the supported typed core through backend-neutral IR and emits executable program.js, Source Map v3 output, package metadata, and kLang-aware runtime errors.

Browser WebAssembly

Compiles the Go interpreter/runtime to klang.wasm and bundles the browser bridge, Go runtime support, resolved kLang sources, and a ready-to-serve page.

Compiler and runtime

The Go engine is split into focused language stages.

1Loadfile or folder workspace
2Resolvelocal and stdlib modules
3Checksemantic and type rules
4Preparereuse parsed program and cache
5Runinterpreter with Args, memory, output
6Packagemanifest, source bundle, WASM files

Source-aware errors

Module, parse, type, runtime, backend, and WASM failures are shaped into ErrorContext records with phase, file, line, column, rule, source line, message, and hint.

Program cache

check and run persist source-fingerprint cache entries in .klang-cache, invalidated by source edits, entry changes, or --raw-lang changes.

State tracking

The checker records compile-time symbol state, and the runtime records define, bind, assign, move, and return transitions exposed through debug_state().

Browser bridge

WASM bundles include klang.wasm, wasm_exec.js, klang_browser.js, index.html, a README, source files, and a klang-build.json manifest.

Standard library

Plain .klang modules extend the language surface.

The stdlib is intentionally small and still evolving. Module resolution can selectively load only referenced functions, while module_caller(call_entire_module : True); opts into complete module loading.

list and array

Safe lookup, copying, transforms, folds, filtering, sorting, and other non-mutating collection operations.

fmt

Runtime-backed Format and Printf wrappers using % placeholders and %% escapes.

strings

Length, prefix/suffix checks, contains, split, join, replace, slice, trim, and defaults.

html

Escaped text, attributes, fragments, full documents, and named browser-oriented HTML tags.

json

Native JSON parsing, validation, serialization, deserialization, typed extraction, and safe Result-based round trips.

option and result

Mapping, chaining, recovery, conversion, fallback, and explicit expectation helpers for fallible programs.

table

Immutable set, delete, merge, ordered views, filtering, mapped values, and fallback composition.

runtime

Worker thread helpers around spawn, join, and thread_status.

hash

Stable hashing helpers for hashable values.

io and test

In-memory readers and writers plus reusable assertions that integrate with the CLI test runner.

metasystem

Program, workspace, build metadata, runtime type information, and source-driven metaprogramming helpers.
